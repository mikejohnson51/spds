The RStudio terminal provides access to the operating system (OS) shell from within the RStudio IDE. Beneath the surface of any OS is an entire world that can only be accessed only from the command line.

The terminal is a gateway to that command line.

With it, instead of pointing and clicking, you can type commands and have your computer respond.

Useful Terminal Commands

In 176A we are only interested in using the terminal for speeding up basic tasks such as creating new files and folders, exploring folder structures, and seeing what it can teach us about our computer system. As such we will introduce only 6 common commands that are useful for navigating a file system and creating files.

pwd print working directory

pwd shows the directory or ‚Äúfolder‚Äù you are currently in.

pwd
## /Users/mikejohnson/github/spds/excersise

mkdir - make a directory

mkdir create a new directory (or folder) in the current working directory. Above we used pwd to see we were in the folder:

## /Users/mikejohnson/github/spds/excersise

If we wanted to add a directory (folder) in that path, we could do so like:

mkdir my-folder
pwd
## mkdir: my-folder: File exists
## /Users/mikejohnson/github/spds/excersise

Notably we see our my-folder directory!

cd change directory

cd allows you to navigate through directories by changing working directory. There are many ways to navigate:

go to subdirectory of current working directory:

pwd
cd my-folder
pwd
## /Users/mikejohnson/github/spds/excersise
## /Users/mikejohnson/github/spds/excersise/my-folder

go to parent (one folder up) of current working directory: cd ..

pwd
cd ..
pwd
## /Users/mikejohnson/github/spds/excersise
## /Users/mikejohnson/github/spds

go to your ‚Äúhome‚Äù directory: cd ~ or simply cd

pwd
cd ~
pwd
## /Users/mikejohnson/github/spds/excersise
## /Users/mikejohnson

Using absolute path, works regardless of your current working directory: cd /home/my_username/Desktop.

pwd
cd /Users/mikejohnson/spds
pwd
## /Users/mikejohnson/github/spds/excersise
## bash: line 1: cd: /Users/mikejohnson/spds: No such file or directory
## /Users/mikejohnson/github/spds/excersise

Windows uses a slightly different syntax with the slashes between the folder names reversed, , e.g.¬†cd C:_USERNAME.

Pro tip 1: Dragging and dropping a file or folder into the terminal window will paste the absolute path into the window. Pro tip 2: Use the tab key to autocomplete unambiguous directory and file names. Hit tab twice to see all ambiguous options.

touch - create a file

Allows you to create a file with a defined name and extension in the working directory. For example, we can create a new R file in our working directory as follows:

touch new-r-file.R

ls list files

ls shows the files in the current working directory. This is equivalent to looking at the files in your Finder/Explorer/File Manager. Use ls -a to also list hidden files, such as .Rhistory and .git. Here we look at all the files in

Telling Git who we are

The first thing we need to do is tell git who we are. This is important because every Git commit uses this information. Git comes with a tool called git config that lets us set the Git configuration variables that control all aspects of how Git looks and operates. To do this, open RStudio and go to the Terminal.

Enter the following, where YOUR NAME is what you want git to know you by (this can be your username or real name)

git config --global user.name 'YOUR NAME'

And tell git the email associated with your GitHub account‚Ä¶

git config --global user.email 'Your Email Here'

When you are done, you can check the configuration of your global options like this:

git config --list --global

Back to GitHub

You now know that Git is a version control software that allows us to track changes in files. You also know GitHub is a web platform for hosting and working with git enabled projects, like the one you are working with! Version control is kind of weird. Not only are you keeping current files, but you are keeping past versions of a file. BUT, you dont want to change each and every little change. Instead, your versions have to be determined by you.

Here is a really helpful analogy for this process:

Imagine you are writing a postcard to your Grandma

You work on the post card to make a nice message, you fix spelling errors, and change a sentence (this is your code). Once you are happy with it you ‚Äústage‚Äù it in you pocket along with a bill, and a package, to take to the post office. When you get the the post office you ‚Äúadd‚Äù all staged to the outgoing mail, The teller commits your mail will go out with a stamp And when the mail goes in to the box it is pushed into the world.

This pattern of modifying, staging, adding, committing and pushing changes is central to a version controled workflow.

Now it is time to introduce the Git tab in RStudio. If you click on the Git tab, you will see all updated files/projects.

  •   To stage files, select the empty box (under ‚ÄòStaged‚Äô) next to every file
  •   Add you files by clicking Commit (a RStudio shortcut bypassing add directly)
  •   Then, commit the staged files by entering a brief commit message (e.g. ‚Äútesting connection‚Äù) and pressing ‚ÄòCommit.‚Äô
  •   After committing, press the upward green arrow to Push. You may need to enter your github username/password to do this.

If you receive no errors along the way then you successfull staged, committed and pushed you changed back to YOUR USERNAME/geog-176A-test
